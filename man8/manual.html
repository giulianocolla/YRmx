NAME

rqacpt - check an exchange for an available message  

SYNOPSIS

#include <yalrt.h>

MSG_DESCRIPTOR *rqacpt ( EXCHANGE_DESCRIPTOR *exch )  

DESCRIPTION

It is a non-blocking call to receive a message from an exchange.

 

RETURN VALUE

If a message is available, then a pointer to that message is returned. if no message is available, a NULL pointer is returned.

 

ERRORS

If the exchange was not properly created, a system error is generated, and Yalrt is frozen.

 

BUGS

None I'm aware of, so far.

rqwait (8), rqsend (8)

NAME

rqctsk - create a task  

SYNOPSIS


void rqctsk(STATIC_TASK_DESCRIPTOR *std )
 

DESCRIPTION

Create a task from the supplied static task descriptor and make it ready.

 

RETURN VALUE

None. If the task created has higher priority than the calling task, then the calling task is preemped, and the newly created task becomes the running task. Otherwise control returns immediately to calling task.

 

ERRORS

If the stack can't be allocated, a system error is generated, and YALRT is frozen.

 

BUGS

No check is performed on priority. A priority of 255, which is forbidden, goes undetected.  

SEE ALSO

rqdtsk(8)

NAME

rqdtsk - delete a task  

SYNOPSIS

#include <yalrt.h>

void rqdtsk( TASK_DESCRIPTOR *task )  

DESCRIPTION

The task is removed from the system, and the allocated stack is made free. An attempt to delete Linux task has no effect. Deleting an already deleted task has no effect.

 

RETURN VALUE

None.

 

ERRORS

None.

 

BUGS

Deleting the running task rqdtsk(rqactv) seems not to work properly. It is advisable to let the task suspend itself rqsusp(rqactv) , and have it deleted later by another task, or by the module cleanup code, which runs in Linux task environment.

NAME

rqresm - resume a suspended task  

SYNOPSIS

#include <yalrt.h>

void rqresm(TASK_DESCRIPTOR *task )  

DESCRIPTION

The task is removed from the suspend list and reintegrated in the ready list. Resuming a task which is not suspended has no effect.

 

RETURN VALUE

None. If the task resumed has higher priority than the calling task, then the calling task is preemped, and the resumed task becomes the running task.

 

ERRORS

None.

NAME

rqsend - send a message to an exchange  

SYNOPSIS

#include <yalrt.h>

void rqsend (EXCHANGE_DESCRIPTOR *exch, MSG_DESCRIPTOR *msg )  

DESCRIPTION

The message is queued to the exchange specified.

 

RETURN VALUE

None. If a task with higher priority than the calling task was waiting at the exchange, then the calling task is preemped, and the receiving task becomes the running task.

 

ERRORS

If the message sent doesn't belong to the task, or if the exchange was not properly created, a system error is generated, and Yalrt is frozen.

 

BUGS

None I'm aware of, so far.

NAME

rqsusp - suspend a task  

SYNOPSIS

#include <yalrt.h>

void rqsusp(TASK_DESCRIPTOR *task )  

DESCRIPTION

If the task is running or ready, it is removed from the ready list, and queued in the suspend list. If the task is waiting, it is suspended as soon as it becomes ready. Suspending an already suspended task has no effect.

 

RETURN VALUE

None. If a task supend itself, the function will return only if the task is resumed by another task. If a task suspends another, control returns to calling task.

 

ERRORS

None.

 

BUGS

None I'm aware of, so far.

NAME

rqwait - receive a message from an exchange  

SYNOPSIS

#include <yalrt.h>

MSG_DESCRIPTOR *rqwait(EXCHANGE_DESCRIPTOR *exch, unsigned short delay )  

DESCRIPTION

It is a blocking call to receive a message from an exchange. A non-zero delay specifies the amount of time in system tick units the task is willing to wait. A zero value means no limit. Waiting at an exchange where no other task is sending messages provides one-shot delay or periodic scheduling functionality.

 

RETURN VALUE

If a message is available, then a pointer to that message is returned immediately. If no message is available, the task is put in waiting state, and the next task in the ready list becomes the running task. When a message becomes available, the task is put again on the ready list, and it receives a pointer to the message. If a time was specified, and the time expires without a message being available, a pointer to a system message with type timeout_type is returned.

 

ERRORS

If the exchange was not properly created, a system error is generated, and Yalrt is frozen.

 

BUGS

None I'm aware of, so far.  

SEE ALSO

rqacpt (8), rqsend (8)
